## Network Access

**The App has no internet permission and makes zero network requests.**

The App does not include the `android.permission.INTERNET` permission in its manifest. It is technically impossible for the App to transmit any data to any server, third party, or external service. All operations occur entirely on your device.

---

## Data Storage

All data created or used by the App is stored exclusively on your device:

- **Master password**: Stored as a cryptographic hash (PBKDF2-HMAC-SHA512 with 210,000 iterations and a random 256-bit salt) in Android's EncryptedSharedPreferences. Your actual password is never stored.
- **Operation history**: Encrypted locally using Android Keystore-backed encryption. Contains only file names, operation types, timestamps, and file sizes. Limited to the last 50 entries.
- **Encrypted files**: Saved to locations you explicitly choose through Android's Storage Access Framework (SAF). The App does not have broad storage access.
- **Biometric data**: The App does not store any biometric data. Biometric authentication is handled entirely by Android's BiometricPrompt API through hardware-backed security (TEE/StrongBox). The App only stores an encrypted verification token in Android Keystore.

---

## Encryption Details

Files are encrypted using:

- **Algorithm**: AES-256-GCM (Galois/Counter Mode)
- **Key Derivation**: PBKDF2-HMAC-SHA512 with 600,000 iterations
- **Salt**: 256-bit, unique per file, generated by `java.security.SecureRandom`
- **IV/Nonce**: 96-bit, unique per chunk, counter-based construction
- **Authentication**: 128-bit GCM tag per 64KB chunk

Your encryption password is used to derive the encryption key and is immediately zeroed from memory after use. The App never stores your file encryption passwords.

## Security

The App implements enterprise-grade security measures:

- AES-256-GCM authenticated encryption with chunked verification
- Android Keystore with hardware-backed security (StrongBox/TEE)
- EncryptedSharedPreferences for all local metadata
- Screenshot and screen recording prevention (FLAG_SECURE)
- Cloud backup disabled (allowBackup=false)
- Exponential brute-force lockout (up to 1 hour)
- Automatic lock on app background timeout
- ProGuard code obfuscation with debug log stripping
- No internet permission — zero remote attack surface

## Data Deletion

You can delete all App data at any time:

1. **Within the App**: Settings → Reset Application — deletes master password hash, operation history, Keystore entries, and all App metadata
2. **Via Android Settings**: Settings → Apps → AES-256 FileVault → Clear Data
3. **Uninstalling the App** removes all App data from your device

Note: Encrypted `.vault` files you created are saved in locations you chose and are not affected by App data deletion. You will need the original encryption password to decrypt them.
